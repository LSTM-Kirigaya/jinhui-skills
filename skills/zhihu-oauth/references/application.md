# 申请开通与测试环境

## 申请流程

当前知乎 OAuth 服务需通过邮件申请开通，步骤如下：

1. 发送申请邮件至 `product-platform@zhihu.com`
2. 等待知乎官方审核并创建应用
3. 获取 `app_id` 和 `app_key`，妥善保存

## 邮件格式

- **收件人**：`product-platform@zhihu.com`
- **主题**：`<公司名称>申请接入知乎 oauth 服务`
- **正文需包含**：
  - 公司名称
  - 应用名称
  - 应用简介
  - `redirect_uri`（授权回调地址，需与申请时一致）

## 申请参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `redirect_uri` | string | 是 | 授权回调地址，生产环境必须使用 HTTPS |

## 权限说明

- 基础用户信息（`uid`、`fullname`、`gender`、`headline`、`avatar_path` 等）默认返回
- `email` 和 `phone_no` 字段需要在申请时向知乎申请相应权限，用户授权后才返回
- 若用户未绑定邮箱或手机号，即使授权了相应权限，响应中也不会包含该字段

## 安全提醒

- `app_key` 属于机密信息，必须存储在服务端，通过环境变量注入
- 严禁将 `app_key` 硬编码在前端代码或提交到版本仓库
